# МОЗАИКА — мобильное приложение (Expo / React Native / TypeScript)

Полная сборка всех экранов из дизайн-документа + анимированный фон-мозаика
(плитки ходят волной, переливаются, бегущий блик).

## Экраны (все подключены в навигацию)
1. **Register** — регистрация (имя, email, пароль, страна — модалка выбора,
   дата рождения, юзернейм с live-проверкой, аватар)
2. **Login** — вход + соцкнопки (VK/MAX/Google — заглушки)
3. **ForgotPassword** — восстановление пароля
4. **TwoFactor** — двухфакторная авторизация (6-значный код)
5. **Onboarding** — карусель из 5 слайдов (мозаика профиля, форматы,
   карта мира, чаты, безопасность)
6. **ProfileSetup** — мастер из 5 шагов (аватар → имя/описание →
   интересы → местоположение → уведомления)
7. **Home** — лента-мозаика: плитки разного размера (2×2, 1×1, широкая),
   верхняя панель с поиском/сообщениями/уведомлениями/созданием поста

## Анимированный фон
`src/components/MosaicBackground.tsx` — сетка плиток-ромбов на
`react-native-reanimated`: каждая плитка колеблется по синусоиде со своей
фазой (получается диагональная волна/бугор), плюс поверх бежит
полупрозрачный блик (LinearGradient, движется по диагонали по кругу).
Используется на всех экранах через `ScreenWrapper` (Register, Login и т.д.)
и напрямую на `HomeScreen`.

Если анимация будет тяжеловата на слабых Android-телефонах — первое, что
стоит уменьшить: количество плиток (`TILE` в MosaicBackground.tsx, сейчас
46px — можно увеличить до 60-70px, плиток станет меньше).

## Важно перед запуском
Вставь свои ключи Supabase в `src/lib/supabase.ts`:
```
SUPABASE_URL = 'https://YOUR_PROJECT.supabase.co'
SUPABASE_ANON_KEY = 'YOUR_ANON_KEY'
```

## Как загрузить и собрать без локального терминала
1. Загрузи содержимое папки как репозиторий на GitHub
   (например, `mosaika-mobile` в организации `nazrOS-softhard`).
2. На https://expo.dev → New Project → Import from GitHub → выбери репозиторий.
3. На странице проекта: **Builds → Create a build** — выбираешь платформу
   (iOS/Android) и профиль сборки, EAS соберёт APK/IPA в облаке.
4. Для живого превью на телефоне без сборки — Development Build через
   Expo Go (QR-код генерируется на dev-сервере; если совсем без
   терминала — можно попросить EAS Build собрать dev-client один раз).

## Что дальше (логичные следующие шаги)
- Экран "Галерея автора" — жест долгого нажатия на карточке ленты,
  двумерная навигация (вертикаль — лента автора, горизонталь — фото),
  описан в документе отдельно
- Профиль пользователя (публичный вид + рейтинг доверия + связи/коллеги/команда)
- Коллекции с обложками
- Моменты (истории, исчезают за сутки)
- Реальная схема БД в Supabase (posts, profiles, collections, moments,
  connections) + Row Level Security
- Единый бэкенд с веб-версией cybereden.ru, если платформы должны шарить
  пользователей/данные

## Галерея автора (жест долгого нажатия)
Реализовано в `src/screens/AuthorGalleryScreen.tsx`. Долгое нажатие (350мс) на
плитке в ленте (`HomeScreen.tsx`) → лёгкая вибрация (`expo-haptics`) →
открывается полноэкранная модалка со всеми публикациями этого автора:
- горизонтальный свайп — переключение между публикациями автора (соседние
  карточки выглядывают по краям на ~4%, как описано в документе)
- свайп вниз — закрывает галерею (drag за палец через PanResponder,
  фон подсвечивается/гаснет пропорционально свайпу)
- крестик или системная кнопка "назад" — тоже закрывают

Подключено в `App.tsx` как модальный экран (`presentation: 'transparentModal'`).

## База данных (Supabase)
`supabase/schema.sql` — полная схема:
- `profiles` (с авто-созданием через триггер при регистрации в auth.users)
- `posts` (плитки мозаики: тип, теги, локация, размер плитки, счётчики)
- `collections` + `collection_posts`
- `moments` (истории с `expires_at`, автоматически "протухают" через RLS-условие)
- `comments` (древовидные — через `parent_id`)
- `likes`, `saves`
- `connections` (Связи/Коллеги/Команда/Поддерживают/Следят — `relation_type`)
- `notifications`
- Storage buckets: `avatars`, `post-media`, `moments-media` (публичное чтение,
  запись только авторизованным)
- Везде включён Row Level Security с политиками "читать может любой,
  писать/удалять — только владелец"

Чтобы применить: Supabase Dashboard → SQL Editor → вставить содержимое
`supabase/schema.sql` → Run.

Соответствующие TypeScript-типы — в `src/lib/database.types.ts`, чтобы
запросы через `supabase-js` (`supabase.from('posts').select()` и т.д.)
были типобезопасными.

## Что осталось мокапом (сознательно, для скорости)
- `src/lib/mockData.ts` — лента сейчас на статичных данных, а не из Supabase.
  Как только применишь `schema.sql`, `HomeScreen`/`AuthorGalleryScreen`
  нужно переключить на `supabase.from('posts').select('*, profiles(*)')`.
- Аватар/фото загружаются не по-настоящему (нет вызова Storage upload) —
  кнопки-заглушки в `RegisterScreen` и `ProfileSetupScreen`.
- Соцвход (VK/MAX/Google) — кнопки без реальной OAuth-логики.

## Полный список экранов (финальная сборка)
Аутентификация: Register, Login, ForgotPassword, TwoFactor.
Онбординг: Onboarding (5 слайдов), ProfileSetup (5 шагов).
Основное приложение:
- **Home** — лента-мозаика, реальные данные из Supabase (`fetchFeedSafe`,
  с фолбэком на моки, если БД ещё не настроена), панель "Моменты" сверху,
  верхняя панель: поиск / карта / сообщения / уведомления / создание поста
- **AuthorGallery** — жест долгого нажатия, галерея публикаций автора
- **Profile** — публичный профиль (свой и чужой), статистика, кнопка "Следить"
- **Search** — поиск людей/публикаций/хэштегов (реальный запрос в Supabase + фолбэк)
- **Messages / Chat** — список диалогов + минималистичный чат
- **Notifications** — лента уведомлений
- **CreatePost** — мастер публикации (7 шагов, реально пишет в Supabase `posts`)
- **Collections** — сетка коллекций пользователя
- **Comments** — древовидные обсуждения (реальные + локальный оптимистичный апдейт)
- **Map** — список публикаций с местоположением (заглушка под реальную карту)

## Что нужно один раз сделать при запуске
1. Накатить `supabase/schema.sql` в SQL Editor проекта Supabase.
2. Вставить `SUPABASE_URL` / `SUPABASE_ANON_KEY` в `src/lib/supabase.ts`.
3. Зарегистрироваться в приложении разок — появится строка в `profiles`.
4. (Опционально) заполнить `supabase/seed.sql` своим `user_id`, чтобы в
   ленте были не только моки, а и настоящие посты.

Если пропустить эти шаги — приложение всё равно полностью открывается и
работает на встроенных моках (`fetchFeedSafe`/`fetchAuthorPostsSafe` сами
падают обратно на `mockData.ts` при любой ошибке или пустом ответе), так
что можно сразу тестировать интерфейс, а бэкенд подключить отдельно.
